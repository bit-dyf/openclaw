# 为 OpenClaw 威胁模型做贡献

感谢您帮助提升 OpenClaw 的安全性。威胁模型是一份持续更新的文档，欢迎任何人参与贡献——您不需要是安全专家。

## 贡献方式

### 添加威胁

发现了我们尚未覆盖的攻击向量或风险？请在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上提交 Issue，用您自己的话描述即可。无需了解任何框架或填写所有字段——只需描述场景。

**提供以下信息会有所帮助（但非必须）：**

- 攻击场景及其利用方式
- OpenClaw 中受影响的部分（CLI、Gateway 网关、渠道、ClawHub、MCP 服务器等）
- 您认为的严重程度（低/中/高/严重）
- 相关研究、CVE 或真实案例的链接

我们将在审查期间处理 ATLAS 映射、威胁 ID 和风险评估。如果您愿意提供这些细节，当然欢迎——但这不是必须的。

> **此处用于补充威胁模型，而非报告已发现的漏洞。** 如果您发现了可利用的漏洞，请查看我们的 [Trust 页面](https://trust.openclaw.ai) 了解负责任披露的说明。

### 建议缓解措施

对现有威胁有改进想法？请提交引用该威胁的 Issue 或 PR。有效的缓解措施应具体且可执行——例如，"在 Gateway 网关层对每个发送者实施每分钟 10 条消息的速率限制"比"实施速率限制"更有价值。

### 提出攻击链

攻击链展示了多个威胁如何组合成现实的攻击场景。如果您发现了危险的组合，请描述步骤以及攻击者如何将其串联起来。一段描述攻击实际展开过程的简短叙述，比正式模板更有价值。

### 修复或改进现有内容

错别字、澄清说明、过时信息、更好的示例——欢迎提交 PR，无需提前创建 Issue。

## 我们使用的框架

### MITRE ATLAS

本威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（AI 系统对抗性威胁图谱）构建，这是一个专门针对 AI/ML 威胁（如提示词注入、工具滥用和智能体利用）设计的框架。贡献时无需了解 ATLAS——我们会在审查期间将提交内容映射到该框架。

### 威胁 ID

每个威胁都有类似 `T-EXEC-003` 的 ID。类别如下：

| 代码    | 类别                          |
| ------- | ----------------------------- |
| RECON   | 侦察——信息收集                |
| ACCESS  | 初始访问——获取入口            |
| EXEC    | 执行——运行恶意操作            |
| PERSIST | 持久化——维持访问              |
| EVADE   | 防御规避——躲避检测            |
| DISC    | 发现——了解环境信息            |
| EXFIL   | 数据渗出——窃取数据            |
| IMPACT  | 影响——造成破坏或中断          |

ID 由维护者在审查期间分配，您无需自行选取。

### 风险等级

| 等级     | 含义                                         |
| -------- | -------------------------------------------- |
| **严重** | 完全系统入侵，或高可能性 + 严重影响          |
| **高**   | 可能造成重大损害，或中等可能性 + 严重影响    |
| **中**   | 中等风险，或低可能性 + 高影响                |
| **低**   | 可能性低且影响有限                           |

如果不确定风险等级，只需描述影响，我们会进行评估。

## 审查流程

1. **分类** — 我们在 48 小时内审查新提交
2. **评估** — 验证可行性，分配 ATLAS 映射和威胁 ID，验证风险等级
3. **文档化** — 确保格式完整
4. **合并** — 添加到威胁模型和可视化图中
